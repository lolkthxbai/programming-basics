# programming-basics
Here, I try to explain common basics of modern programming languages to help you better understand what it means to "code"

## What is code?

What we call "code" is the written words used to communicate with computers, typically instructions for the computer to execute or "run". When we write code in a organized structure that can be run, we call that a "program" or "apps" (short for applications). There is many more processes and systems that are running in the background in order to make all this possible. However, I won't be going into that in this overview. If you finish reading this a find that you're curious and want to learn more I recommend you search Youtube for terms such as "Binary", "Machince Code", "BASIC Code"

So what? We can tell a computer what to do but why does that matter? Well, generally speaking, a computer by definition must capable of these 4 (four) things:

1. Input Data
2. Process Data
3. Store Data
4. Output Data

And it's these 4 (four) things that allow us to create things from our own imagination. It may not be obvious at first but, using a computer to create things with code allows us to be a god in our own virtual world. At least, thats how I see it anyways...

## Data

I hope I don't have to explain what is data but just think of it as any sort of information. Even if doesn't make sense to you, any kind of information is considered data. For example, a recipe book is full of data; It contains information on how to prepare many different types of food. Now, the reciepe book can be written in English or perhaps Spanish, or even Chinese. And unless you know how to read a certain langauge, you won't understand data written in that langauge. The same is true for computers. Computers don't understand English, Spanish or Chinese. They understand binary code, '1's' and '0's'. Like I said, I won't go into detail about binary, but its important you understand that we can't talk to computer chips in the same langauge we speak. So to talk to computers we invented a new language that is easier to understand for us and easy for a computer to translate into binary. Thats what we call 'programming lanugage' and theres a ton of different langauges. In fact, there are about as many programming langauges as there are spoken langauges.
